## **Overview**

This repository contains my final submission for the [Mitsui Commodity Prediction Challenge](https://www.kaggle.com/competitions/mitsui-commodity-prediction-challenge). The goal of the competition was to predict the future price movements of 424 financial targets across various asset classes using historical time-series data.

Given the inherently noisy nature of financial data, my approach focused heavily on preventing look-ahead bias, extracting stable cross-asset signals, and building a highly robust validation pipeline to mitigate temporal overfitting.

---

## **Solution Architecture**

### **1. Leak-Free Feature Engineering**

To prevent data leakage and look-ahead bias, a strict **shift-by-1-period** firewall was applied to all base columns before any rolling mathematics or aggregations occurred. Feature highlights include:

* **Cross-Asset Aggregations:** Moving averages and standard deviations across broad asset classes (LME, JPX, US Stock, FX).
* **Spreads & Ratios:** Z-scored spreads and rolling ratios to capture divergence trading signals (e.g., Gold/Platinum ratio, GLD/SLV spread).
* **Advanced Volatility:** Parkinson volatility and Normalized Average True Range (ATR) utilizing high/low price extremes.
* **Macro Indicators:** Currency carry trade momentum and stress indicators built on JPY pairs.
* **Momentum:** Rate of Change (ROC), Relative Strength Index (RSI), and volume momentum.

### **2. Three-Tier Feature Selection**

Feeding hundreds of financial features into a gradient booster invites severe overfitting. I implemented a reduction strategy to distill the feature set down to the most stable 200 signals:

1. **Adversarial Validation (Stability Pruning):** Trained a classifier to distinguish between train and test sets, dropping the top 15% most unstable features that exhibited temporal drift.
2. **Correlation Pruning (Redundancy Removal):** Dropped features with >0.95 Pearson correlation to reduce extreme collinearity.
3. **Signal Pruning:** Fitted a preliminary LightGBM model on a subset of targets to rank surviving features by predictive importance.

### **3. Validation Strategy**

Standard k-fold cross-validation fails on financial time-series due to overlapping horizons.

* Implemented **Purged Cross-Validation with Embargo** to strictly drop training samples whose forecast windows overlapped with the validation set's timeline.
* Evaluated locally using a custom `rankcorr_sharpe` function to mirror Kaggle's backend metric (Sharpe ratio of daily Spearman rank correlations).

### **4. Modeling & Ensemble**

The final predictions were generated by blending two distinct gradient boosting architectures to diversify algorithmic risk and smooth out extreme predictions:

* **CatBoost Regressor** (60% weight): Leveraged for its oblivious trees and strength against overfitting.
* **LightGBM Regressor** (40% weight): Leveraged for its fast, leaf-wise growth.
* **Engineering Note:** To handle Kaggle's memory constraints and avoid OOM crashes during inference, the pipeline utilizes a lazy-loading mechanism to efficiently manage over 800 discrete model `.pkl` files (424 targets $\times$ 2 models) through the Mitsui Inference Server API.

## **Requirements & Execution**

* `polars`
* `pandas`
* `numpy`
* `lightgbm`
* `catboost`
* `scipy`

The pipeline is governed by a `TRAIN` boolean flag. When set to `True`, it executes feature engineering, purging CV, trains the 848 final models, and saves them. When `False` (or detected to be running in the hidden Kaggle scoring environment), it hooks into the `MitsuiInferenceServer` callback API to engineer batch features and yield real-time ensembled predictions.
